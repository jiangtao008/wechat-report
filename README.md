# 微信群周报

这是一个面向微信群聊记录的 AI 周报项目，核心目标是把原始聊天文本转换成一份可直接打开的 HTML 报告。报告重点不放在流水账式复述，而是聚焦社群动态、讨论节奏、活跃成员和群体情绪。

## 项目定位

这个项目对应的 skill 是 `wechat-group-report`，用于分析微信群聊记录，并生成黑色科技风的日报、周报或月报。输入可以是：

- 直接粘贴的聊天文本
- 本地 `.txt` 导出文件
- 本地 `.md` 聊天转录文件
- 本地 `.csv` 聊天导出文件

输出是完整的 `.html` 文件，可直接在浏览器中打开查看。

## 当前 skill 能力

当前版本的 `wechat-group-report` 已经不是简单“总结聊天记录”，而是更偏社群情报报告的生成器，重点能力包括：

- 根据聊天体量自适应整理热点话题，通常 `3 到 5` 条，内容很多时可扩展，但通常不超过 `10` 条
- 为热点话题生成带过程感的结构化叙述，必要时拆成 `第一幕 / 第二幕 / 第三幕 / 第四幕`
- 对活跃成员生成更具观察力的人物侧写，而不是模板化一句话介绍
- 为活跃成员补充更精准的短标签，标签更像“群内工位名”或“外号”，而不是空泛表扬词
- 使用 `canvas + wordcloud2.js` 渲染关键词云
- 在区间总览中展示微趋势曲线，用来表现聊天热度随时间的起伏

## 周报内容结构

生成后的周报主要包含以下模块：

### 1. 区间总览

用于快速给出这一时间段群聊的整体画像，通常包括：

- 总发言条数
- 活跃参与人数
- 高频情绪
- 微趋势曲线

其中“高频情绪”会单独放一行，用来概括这段时间群聊整体更接近兴奋、调侃、围观、焦虑、捧场还是别的主情绪。

“微趋势曲线”用于展示聊天热度变化，当前 UI 特征包括：

- 横轴为时间，可细化到小时粒度
- 纵轴为聊天条数
- 曲线做平滑处理，避免折线过硬
- 同时展示横向与纵向网格线，便于判断热度起伏

### 2. 群聊热点话题

这是整份周报的核心内容，关注的是“群里到底在热烈聊什么”。

每个热点话题会尽量做成更有叙事感的结构，而不是一段压缩的摘要，通常包括：

- 显眼的话题标题
- 过程化拆解
- 必要时拆成 `第一幕 / 第二幕 / 第三幕 / 第四幕`
- 点睛之笔
- AI 锐评
- 主要参与者

其中：

- 热点话题数量不是固定写死的，会根据聊天条数和讨论密度自适应调整
- 过程内容强调聊天推进顺序，体现谁先抛话题、谁接话、氛围如何演化
- 如果话题本身内容不够丰富，就不强行写“第几幕”，而是直接展开关键过程节点
- `点睛之笔` 优先引用聊天原文，并在右下角标出发言群友名字
- `AI 锐评` 风格偏幽默、锐利、犀利，强调一针见血
- `主要参与者` 放在每个热点话题末尾，帮助快速定位该话题的关键发言人
- 对高讨论度话题，不只保留结论，还会尽量交代讨论是如何一步步推进到该结论上的

### 3. 区间活跃之星

这一部分不只是做发言排行，还会对高频发言成员做风格判断。

每位活跃之星都会包含：

- 成员名称
- 发言次数
- AI 锐评
- 一个或多个标签

活跃之星人数最多支持 20 人，不强制凑满，且发言少于 `10` 条的成员不会进入这一栏。

这一部分的最新要求是：

- AI 锐评优先写成人物侧写，通常 `2 到 4` 句，素材足够时可扩到 `4 到 6` 句
- 文风允许适度幽默、调侃、犀利、带刺、轻微腹黑，但必须基于聊天事实
- 评论重点不是“谁很活跃”，而是“这个人在群里到底扮演什么角色”
- 如果某人高频但信息密度不高，可以直接点出来，但要像高阶吐槽，不能像情绪发泄

标签支持多个，但强调“短且准”，例如：

- `控场手`
- `接梗王`
- `情报源`
- `抛问机`
- `补刀位`
- `气氛泵`
- `链接投手`
- `节奏校准器`

不鼓励使用这类空泛标签：

- `很活跃`
- `爱发言`
- `热心群友`
- `优秀成员`

### 4. 关键词云

关键词云不是静态标签堆，而是实际可视化的 `canvas` 词云。

当前实现方式：

- 使用 `wordcloud2.js`
- 使用带权重的二维数组作为数据源
- 直接在 HTML 中通过 `canvas` 渲染
- 保持黑色科技风视觉风格

关键词会过滤掉低信息量停用词，只保留名词、动词、产品名、事件名和反复出现的核心概念。

## 主要功能

这个项目当前已经具备的主要能力包括：

- 从微信群聊记录中识别讨论最集中的热点话题
- 根据聊天密度自适应决定热点数量和展开程度
- 统计发言次数并生成人物活跃榜
- 为活跃成员生成更有观察力的人物侧写和精准标签
- 总结群聊主情绪并放入总览区
- 在总览区生成聊天热度微趋势曲线
- 为热点话题生成带过程感、可分幕的分析结构
- 生成基于 `canvas` 的关键词云
- 输出黑色科技风、可直接浏览的 HTML 报告

## 输出风格特点

这份周报不是传统纪要，而是偏“社群情报报告”风格，特点包括：

- 黑色科技风 UI
- 强结构化阅读体验
- 重点模块突出显示
- 热点标题更显眼
- 过程感强，适合快速扫读
- 文案带有 AI 分析视角，而不是简单摘抄原文
- 活跃之星文案更像人物小传，而不是荣誉榜播报
- 话题点评允许更有戏谑感，但不脱离聊天事实
- 区间总览不只给静态数字，还会补热度走势

## 适用场景

适合用于：

- 微信群日报
- 微信群周报
- 微信群月报
- 社群运营复盘
- 群成员活跃观察
- 热点话题沉淀

## 当前实现说明

周报的生成逻辑和约束主要来自 skill `wechat-group-report`，其能力集中在：

- 输入识别
- 报告范围判定
- 热点话题提炼
- 热点话题过程化展开与分幕控制
- 活跃成员画像
- 活跃成员标签精炼
- 微趋势曲线展示
- 关键词云数据组织
- HTML 模板输出

如果后续还要继续扩展，可以优先考虑：

- 自动读取聊天原始文件并批量出报告
- 增加日报 / 周报 / 月报切换策略
- 为总览区补更多统计项
- 增加更丰富的情绪分析维度
