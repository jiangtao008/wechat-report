# 支持的输入格式

当传入的微信群聊记录格式混乱，或缺少明确结构时，读取这份参考。

## 优先支持的输入

- 对话中直接粘贴的原始聊天文本
- 本地 `.txt` 导出文件
- 本地 `.md` 聊天转录文件
- 包含发言人与消息列的本地 `.csv` 导出文件

## 常见格式

### 纯文本聊天记录

常见形式包括：

- `2026-05-27 09:15 张三：消息内容`
- `张三 09:15`
  后面跟随一行或多行消息内容

解析纯文本时：

- 如果没有时间戳，来自同一发言人的连续多行内容可视为同一条消息块
- 金句部分尽量保留原始引用措辞
- 根据可见时间戳推断日期范围

### CSV 导出文件

优先寻找含义接近下面这些的列：

- time / timestamp / date
- speaker / sender / user / nickname
- content / message / text

如果列名不完全一致，先按语义映射字段，再开始分析。

## 清洗规则

- 忽略没有语义价值的媒体占位符，例如 `[图片]`、`[表情]`、`[语音]`，除非它们本身就是讨论重点。
- 忽略类似 `你已添加了某某，现在可以开始聊天了` 这样的系统提示。
- 保留链接、产品名、活动名和重复出现的专有名词，因为它们通常是有价值的关键词。
- 对转发或重复出现的内容，默认视为同一讨论信号；只有当“重复”本身有意义时，再单独看待。

## 兜底规则

- 如果缺少群名，使用 `微信群`。
- 如果日期范围不完整，就如实写出当前可见区间。
- 如果参与者身份存在歧义，直接使用聊天记录中原始可见的标识进行排序，不要自行编造名称。
